FAQ
Common questions
Is my code stored on your servers?
No. ZIP files are extracted to a temporary directory, scanned, and the directory is deleted immediately after. GitHub repos are cloned at depth=1 and removed right after scanning. We never persist your source code.
What's the difference between free and Pro?
Free users get 1 scan per day and see only the first 5 findings. Pro users get unlimited scans, see every finding with full detail, and can download a PDF report for compliance or sharing with your team.
How does detection work? Are there false positives?
We use 25+ compiled regex patterns, each matched to a specific secret type. Patterns also check context to avoid matching placeholder values like "your-api-key-here". False positive rates are very low.
What file types are scanned?
We scan .py, .js, .ts, .env, .yaml, .json, .toml, .sh, .php, .go, .java, .tf, .properties, .pem, .key, and many more. Binary files, images, and minified bundles are automatically skipped.
I found a secret in my repo. What do I do?
Each finding includes a specific remediation step. Rotate the secret immediately at the provider's dashboard, then remove it from your codebase and git history using git-filter-repo or BFG Repo Cleaner.
